## 功能使用

### 1. 预约会议室

**步骤：**

1. 选择菜单选项 `1`
2. 输入组织者姓名
3. 输入会议室名称
4. 输入会议主题
5. 输入开始时间（格式：YYYY-MM-DD HH:MM）
6. 输入结束时间（格式：YYYY-MM-DD HH:MM）
7. 输入参与人数

**示例：**

```
============================================================
预约会议室
============================================================
组织者姓名: 张三
会议室名称: A会议室
会议主题: 项目讨论

时间格式示例: 2026-04-01 09:00
开始时间: 2026-04-01 09:00
结束时间: 2026-04-01 11:00
参与人数: 5

[成功] 预约成功，会议ID: 1
```

**注意事项：**

- 时间格式必须是 `YYYY-MM-DD HH:MM`
- 结束时间必须晚于开始时间
- 开始时间不能早于当前时间
- 同一会议室同一时间段不能重复预约

## 错误处理

### 常见错误及解决方案

#### 1. 无法连接到服务端

**错误信息：**
```
无法连接到服务端
```

**解决方案：**
- 确保服务端已启动
- 检查服务端是否在 `127.0.0.1:8000` 监听
- 检查防火墙设置

#### 2. 时间格式错误

**错误信息：**
```
错误: 时间格式不正确，请使用格式: YYYY-MM-DD HH:MM
```

**解决方案：**
- 使用正确的格式，如：`2026-04-01 09:00`
- 注意日期和时间中间有空格
- 时间使用24小时制

#### 3. 时间逻辑错误

**错误信息：**
```
错误: 结束时间必须晚于开始时间
```

**解决方案：**
- 确保结束时间晚于开始时间

**错误信息：**
```
错误: 开始时间不能早于当前时间
```

**解决方案：**
- 开始时间必须是未来的时间

#### 4. 时间冲突

**错误信息：**
```
[失败] 该会议室在指定时间段已被预约
```

**解决方案：**
- 选择其他时间段
- 选择其他会议室

#### 5. 会议不存在

**错误信息：**
```
[失败] 未找到会议ID为 999 的预约
```

**解决方案：**
- 检查输入的会议ID是否正确
- 使用查询功能确认会议ID

#### 6. 重复取消

**错误信息：**
```
[失败] 会议已被取消
```

**解决方案：**
- 该会议已被取消，无需再次操作

## 数据管理

### 数据库文件

- **格式**: SQLite数据库

## 技术说明

### RPC通信

- **协议**: HTTP/XML-RPC
- **端口**: 8000
- **地址**: 127.0.0.1 (localhost)
- **序列化**: XML

### 数据库

- **类型**: SQLite
- **连接**: Python内置sqlite3模块
- **索引**: 自动创建以提高查询效率

### 安全性

- 使用参数化查询防止SQL注入
- 输入验证防止非法数据
- 时间验证防止时间逻辑错误

## 故障排除

### 端口被占用

**症状：** 服务端启动失败

**解决方案：**
```bash
# 查找占用8000端口的进程
netstat -ano | findstr :8000

# 结束进程或修改代码中的端口号
```

### 数据库锁定

**症状：** 数据库操作失败

**解决方案：**
- 确保只有一个服务端实例在运行
- 检查数据库文件是否被其他程序打开
